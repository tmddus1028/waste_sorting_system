# 3. Class Analysis

## 3.1 Class Description

| Class Name | Description |
| :--- | :--- |
| **User** | User 클래스는 시스템을 이용하는 일반 사용자를 의미한다. 사용자는 회원가입과 로그인을 통해 시스템에 접속하며 쓰레기 이미지를 업로드하거나 재활용 정보를 검색할 수 있다. 또한 식별된 쓰레기 품목에 대한 분리배출 가이드와 지역별 분리배출 규칙을 확인할 수 있다. |
| **Administrator** | Administrator 클래스는 시스템의 데이터를 관리하는 관리자를 의미한다. 관리자는 일반 사용자와 달리 재활용 정보, 분리배출 방법, 지역별 규칙 데이터를 추가하거나 수정할 수 있다. 이 클래스는 시스템 정보가 최신 상태로 유지되도록 관리하는 역할을 한다. |
| **Account** | Account 클래스는 사용자와 관리자의 계정 정보를 관리한다. 로그인에 필요한 이메일, 아이디, 비밀번호, 권한 정보를 포함한다. User와 Administrator는 Account 정보를 통해 시스템에 인증된다. |
| **WasteImage** | WasteImage 클래스는 사용자가 업로드한 쓰레기 사진을 의미한다. 시스템은 이 이미지를 분석하여 쓰레기 종류를 식별한다. 이미지 파일명, 저장 위치, 파일 형식, 업로드 시간 등의 정보를 포함할 수 있다. |
| **WasteItem** | WasteItem 클래스는 이미지 인식 결과로 식별된 쓰레기 품목을 의미한다. 예를 들어 플라스틱, 종이, 캔, 유리, 음식물 쓰레기 등이 WasteItem에 해당한다. 이 클래스는 RecyclingGuide와 연결되어 해당 품목의 분리배출 방법을 제공하는 기준이 된다. |
| **ImageRecognitionResult** | ImageRecognitionResult 클래스는 이미지 분석 결과를 저장한다. 시스템이 어떤 쓰레기 품목으로 판단했는지, 분석 정확도는 어느 정도인지, 분석이 성공했는지 실패했는지를 포함한다. 분석 실패 시 재업로드 또는 수동 검색으로 연결될 수 있다. |
| **RecyclingGuide** | RecyclingGuide 클래스는 쓰레기 품목별 분리배출 방법을 제공한다. 해당 품목이 재활용 가능한지, 어떻게 배출해야 하는지, 세척이 필요한지, 라벨이나 뚜껑을 분리해야 하는지 등의 정보를 포함한다. |
| **DetailedGuide** | DetailedGuide 클래스는 기본 분리배출 안내보다 더 구체적인 정보를 제공한다. 예를 들어 오염된 플라스틱 처리 방법, 종이컵 코팅 여부에 따른 배출 방법, 유리병과 깨진 유리의 차이 등을 설명한다. |
| **LocalRecyclingRule** | LocalRecyclingRule 클래스는 지역별로 다른 분리배출 규칙을 관리한다. 지역명, 수거 요일, 배출 시간, 배출 장소, 대형 폐기물 처리 방식 등의 정보를 포함한다. 사용자는 자신의 지역을 선택하여 해당 규칙을 확인할 수 있다. |
| **RecyclingInformation** | RecyclingInformation 클래스는 사용자가 검색할 수 있는 재활용 관련 정보를 의미한다. 사용자가 특정 품목명을 검색하면 시스템은 이 클래스의 정보를 바탕으로 재활용 가능 여부와 처리 방법을 제공한다. |
| **SearchKeyword** | SearchKeyword 클래스는 사용자가 검색 기능을 이용할 때 입력한 검색어를 의미한다. 검색어, 검색 시간, 검색 결과와 연결될 수 있으며, 사용자의 검색 기록을 관리하는 데 활용될 수 있다. |
| **RecyclingData** | RecyclingData 클래스는 시스템에서 사용하는 재활용 관련 데이터 전체를 의미한다. 품목별 재활용 정보, 분리배출 가이드, 지역별 규칙 등이 포함된다. 관리자는 이 데이터를 수정하거나 추가할 수 있다. |
| **UploadHistory** | UploadHistory 클래스는 사용자의 이미지 업로드 기록을 저장한다. 어떤 사용자가 어떤 이미지를 언제 업로드했는지, 그 이미지의 분석 결과가 무엇이었는지를 관리한다. |
| **SystemServer** | SystemServer 클래스는 사용자의 요청을 처리하는 서버를 의미한다. 이미지 업로드 요청, 로그인 요청, 재활용 정보 검색 요청, 지역별 규칙 조회 요청 등을 처리하고 필요한 결과를 사용자에게 반환한다. |
| **Database** | Database 클래스는 시스템 운영에 필요한 데이터를 저장하는 클래스이다. 사용자 계정 정보, 쓰레기 이미지 정보, 분석 결과, 재활용 가이드, 지역별 규칙 등이 데이터베이스에 저장된다. |
